תיעוד API -פעולוות מנהל

CRUD כתובת פרמטרים ב־URL גוף הבקשה תיאור הפעולה מה מחזירה הפונקציה שגיאות אפשריות
GET /api/admin/unapprove – – מחזירה את כל המשתמשים עם סטטוס unapprove 200 OK: { title, users }
404: אין משתמשים		 404 – אין משתמשים
500 – שגיאה כללית
GET /api/admin/waiting – – מחזירה את כל המשתמשים עם סטטוס waiting 200 OK: { title, users }
404: אין משתמשים		 404 – אין משתמשים
500 – שגיאה כללית
PUT /api/admin/update/:userId/status userId { "status": "approve" | "unapprove" | "waiting" } מעדכנת סטטוס של משתמש ע"י אדמין 200 OK: { message, user } 400 – ID לא תקין
400 – סטטוס לא חוקי
400 – כבר באותו סטטוס
404 – משתמש לא נמצא
PUT /api/admin/unapprove/:userId userId – משנה סטטוס של יועץ ל־unapprove 200 OK: { message, user } 400 – ID לא תקין
400 – כבר לא מאושר
404 – משתמש לא נמצא
500 – שגיאה כללית
PUT /api/admin/approve/:userId userId – מאשר יועץ – משנה סטטוס ל־approve 200 OK: { title, user } 400 – ID לא תקין
400 – כבר מאושר
404 – משתמש לא נמצא
500 – שגיאה כללית
